Efficient Frontier

PART 1: MAIN GUIDE

1. Identity

Full Name: Efficient Frontier

Abbreviation: EF

Category: Portfolio Optimization

Creator: Harry Markowitz (1952)

One-Sentence Definition: The Efficient Frontier is the set of optimal portfolios that offer the highest expected return for a given level of risk, forming the upper boundary of all possible portfolio combinations.

2. Calculation Logic

Formula

For a portfolio with weights $\mathbf{w}$:

$$\text{Portfolio Return: } \mu_p = \mathbf{w}^T \mathbf{r}$$

$$\text{Portfolio Risk: } \sigma_p = \sqrt{\mathbf{w}^T \Sigma \mathbf{w}}$$

Optimization Problem:

$$\min_{\mathbf{w}} \mathbf{w}^T \Sigma \mathbf{w}$$

Subject to: $\mathbf{w}^T \mathbf{r} = \mu_{target}$ and $\sum w_i = 1$

StepDescription
1Collect expected returns and covariance matrix
2For each target return level, solve minimum variance optimization
3Plot risk-return pairs to form the frontier curve
4Tangent line from risk-free rate identifies maximum Sharpe portfolio

Real-World Analogy

Think of the Efficient Frontier like a budget airline route map. You want to travel far (high return) cheaply (low risk). The frontier shows which destinations are achievable at each price point. Anything "inside" the frontier is inefficient—paying more for the same trip.

7. Practical Cautions

Estimation Error: Small changes in expected returns cause large weight swings. Use shrinkage estimators.
Infeasible Solutions: If target return > max asset return, optimization fails.

8. Pro Insights

The Northwest Rule: On the risk-return scatter, always aim "Northwest" (up and left). Any portfolio to your Southeast is strictly dominated—same return, more risk.

Frontier Instability: The curve "dances" with each data update. Treat it as a probabilistic zone, not a fixed line.

9. Summary

"The Efficient Frontier maps the best possible portfolios—everything inside the curve is leaving money on the table."

PART 2: SUPPLEMENTARY GUIDE

1. Core Concept

The Efficient Frontier answers: "What's the maximum return I can get for a given risk level?"

Before Markowitz, investors chose assets individually. The frontier proved that *combinations* of risky assets can produce portfolios with better risk-return profiles than any individual asset.

2. Statistical Edge

Mean-variance optimized portfolios have historically outperformed equal-weight portfolios on a risk-adjusted basis when inputs are reasonably estimated.

3. Actionable Strategy: "Momentum-Filtered MVO"


STEP 1: Rank assets by 6-month momentum
STEP 2: Keep top 50% (filter losers)
STEP 3: Calculate efficient frontier for winners only
STEP 4: Select tangency portfolio (max Sharpe)
STEP 5: Rebalance monthly

4. Risks & Limitations

- Corner solutions: Optimizer concentrates in 1-2 assets

- Estimation error: Garbage in, garbage out

- Singular matrix: More assets than observations

5. Further Study

Signal Patterns

Efficient Frontier Check

This function evaluates whether a current portfolio allocation lies on or near the efficient frontier. Portfolios inside the curve are suboptimal—achieving the same return with more risk, or the same risk with less return. A 5% tolerance accounts for estimation uncertainty and transaction costs inherent in optimization.


def on_efficient_frontier(portfolio_return, portfolio_risk, frontier_points):
    for (f_ret, f_risk) in frontier_points:
        if abs(f_ret - portfolio_return) < 0.001:
            return portfolio_risk <= f_risk * 1.05  # Within 5% of frontier
    return False
