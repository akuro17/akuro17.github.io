Renko Charts

PART 1: MAIN GUIDE

1. Identity

Full Name: Renko Charts (Renko-ashi)

Abbreviation: N/A (Chart Type)

Category: Japanese Structural / Noise Filtration

Creator: Traditional Japanese (19th Century); Introduced to West by Steve Nison (~1991)

One-Sentence Definition: Renko is a time-independent charting method that plots "bricks" only when price moves by a predefined amount, filtering out market noise to reveal pure trend structure.

2. Calculation Logic

Formula

Renko uses algorithmic logic rather than a static formula, governed by a Box Size ($B$):

Up Brick Generation:

$$\text{If } P_t \ge P_{top} + B$$

$$\text{New High} = P_{top} + B \times \lfloor \frac{P_t - P_{top}}{B} \rfloor$$

Down Brick Generation:

$$\text{If } P_t \le P_{bottom} - B$$

$$\text{New Low} = P_{bottom} - B \times \lfloor \frac{P_{bottom} - P_t}{B} \rfloor$$

StepLogicPurpose
1Initialize currentLevel = Floor(Close / BrickSize) × BrickSizeEstablish baseline
2For each candle, check if High ≥ currentLevel + BrickSizeUp brick trigger
3If triggered, create Up brick, increment currentLevelBuild up brick
4Check if Low ≤ currentLevel - BrickSizeDown brick trigger
5If triggered, create Down brick, decrement currentLevelBuild down brick

Real-World Analogy

Think of Renko like a hotel elevator display—it only shows floor changes when you actually arrive at a new floor. The time between floors is irrelevant; what matters is whether you've moved up or down by a complete floor. Small movements within a floor are ignored as noise.

Why These Values?

ATR-based box sizes automatically adjust to market volatility, ensuring consistent brick formation frequency across different market conditions. This prevents charts from becoming too noisy in high volatility or too sparse in low volatility.

7. Practical Cautions

Lag Characteristics

ComponentApproximate Lag
First Reversal Signal2× Box Size price movement
Confirmation (2-brick)3× Box Size price movement
Brick FormationVariable (depends on volatility)
Reality Check: A reversal requires price to traverse the entire previous brick, then form a new one in the opposite direction. This inherent lag means you'll miss the exact bottom/top.

False Signal Scenarios

1. The Single-Brick Trap

A single reversal brick forms, then the original trend resumes. Wait for 2 consecutive bricks.

2. Gap-Induced Phantoms

Price gaps create multiple "phantom" bricks from a single candle. These aren't real price action.

3. ATR Repainting

Dynamic box sizes based on ATR can cause historical chart changes as ATR values update.

Mitigation Strategies

- Two-Brick Rule: Wait for 2 consecutive reversal bricks before entering

- Fixed Box Size: Use fixed sizes for historical consistency

- Moving Average Filter: Only trade with the MA trend

8. Pro Insights

The Wick Revelation

Standard Renko hides wicks, but High/Low Renko variants display them. Long rejection wicks on reversal bricks reveal hidden exhaustion—price had the volatility but lacked conviction to hold.

Pattern Reliability Advantage

Chart patterns (Double Tops, Head & Shoulders) are significantly more reliable on Renko charts than standard candlesticks because each brick represents a substantial, volatility-filtered price movement—not random intraday noise.

The Brick & SuperTrend Lock

Calculate SuperTrend using the same ATR as Renko box size. This creates perfect synchronization:

- Entry: Green brick + Price > SuperTrend

- Exit: Price < SuperTrend (not on single red brick)

This prevents getting shaken out by single reversal bricks while riding massive trends.

9. Summary

"Renko charts eliminate time to reveal what really matters—whether price actually moved or just vibrated in place."

PART 2: SUPPLEMENTARY GUIDE

1. Core Concept

Renko answers one question: "Has price moved enough to matter?"

Unlike candlesticks that plot every minute/hour regardless of movement, Renko only creates a new brick when price travels a meaningful distance. This is like tracking a hiker's progress by milestones rather than by the clock—if they rest for an hour, no new milestone is recorded.

The name comes from the Japanese word *renga* (磚), meaning "brick." Each brick represents a completed unit of price movement, stacked like masonry to form visual trend structure.

2. Statistical Edge

Backtest Evidence

Pattern Reliability~55%~70%

Why It Works

Renko's edge is information compression. By requiring price to move a full box size before plotting, noise is systematically removed. What remains is the essential signal—price direction that overcame the volatility threshold.

Historical Note

Japanese rice merchants developed this technique centuries before computers, understanding intuitively that daily price recordings were often meaningless noise. Only moves that exceeded their "box" (likely a unit of rice quantity or yen) were worth noting.

3. Actionable Strategy

Strategy Name: "Brick & Trend Lock"

Concept: Ride trends using Renko for direction and SuperTrend for stop placement.

Setup Requirements:

1. Calculate Box Size = ATR(14)

2. Apply SuperTrend with same ATR(14)

Entry Trigger:

- First Green (Up) brick forms after Red series

- Brick Close > SuperTrend line

Position Sizing:

- Standard size when brick count in direction = 1-3

- Reduce size when brick count > 10 (extended trend)

Exit Rules:

1. Primary Exit: Price closes below SuperTrend

2. Alternative Exit: 3 consecutive reversal bricks

3. Time Stop: None (Renko is time-independent)

Example:


Stock XYZ shows:
  3 Red bricks form (downtrend)
  First Green brick forms, closes at $105
  SuperTrend shows green line at $102
  
  Entry: $105 (Green brick close > $102 SuperTrend)
  Stop: $102 (SuperTrend level)
  
  Trend continues: 8 more green bricks
  SuperTrend rises to $115
  Red brick forms, closes at $118
  Still above SuperTrend → HOLD

Psychological Traps

1. The "I Missed It" Trap

Seeing a string of same-color bricks can create FOMO. Remember: late entry is better than wrong entry.

2. Brick Count Obsession

Counting bricks to predict reversals is futile. Bricks count movement, not time or probability.

3. Over-Optimization

Constantly adjusting box size destroys consistency. Choose a method and stick with it.

Common Misinterpretations

- "More bricks = stronger trend" → More bricks = more movement, not necessarily sustainable

- "Gap bricks are real signals" → Gaps create artificial chart distortions

- "Renko shows exact entry prices" → Brick edges are thresholds, not precise levels

Signal Patterns

Buy Signal (Trend Reversal)

This signal triggers when a green (up) brick forms after a series of red (down) bricks, indicating price has moved enough in the opposite direction to form a reversal brick. The requirement for multiple prior down bricks ensures you're trading a genuine trend reversal, not noise. Waiting for 2 consecutive reversal bricks before entry dramatically improves reliability.


def renko_buy_signal(bricks):
    condition_1 = bricks[-1].BrickType == "Up"           # Current is green
    condition_2 = bricks[-2].BrickType == "Down"         # Previous was red
    condition_3 = len([b for b in bricks[-5:] if b.BrickType == "Down"]) >= 2  # Downtrend existed
    
    return all([condition_1, condition_2, condition_3])

Sell Signal (Trend Reversal)

This exit signal fires when a red (down) brick forms after a series of green (up) bricks, confirming price has reversed and overcome the 2x box size threshold required for a reversal brick. The inherent lag of Renko (requiring full box size + reversal) means you'll miss the exact top, but this is the cost of noise filtration. Chart patterns are ~40% more reliable on Renko than standard candlesticks due to this filtering.


def renko_sell_signal(bricks):
    condition_1 = bricks[-1].BrickType == "Down"         # Current is red
    condition_2 = bricks[-2].BrickType == "Up"           # Previous was green
    condition_3 = len([b for b in bricks[-5:] if b.BrickType == "Up"]) >= 2  # Uptrend existed
    
    return all([condition_1, condition_2, condition_3])

Double Bottom Pattern

The classic "W" pattern is highly reliable on Renko charts because each brick represents a substantial, volatility-filtered price movement. This function identifies two down brick lows at similar price levels (within 0.5× brick size) followed by an up brick. Double bottom patterns on Renko have ~70% reliability compared to ~55% on standard candlesticks.


def renko_double_bottom(bricks, price_tolerance=0.5):
    # Find two Down brick lows at similar price levels
    # with Green bricks between and after
    lows = [(i, b.Close) for i, b in enumerate(bricks) if b.BrickType == "Down"]
    
    for i, (idx1, low1) in enumerate(lows[:-1]):
        for idx2, low2 in lows[i+1:]:
            if abs(low1 - low2) <= price_tolerance * brick_size:
                # Check for green bricks after second low
                if bricks[-1].BrickType == "Up":
                    return True
    return False
